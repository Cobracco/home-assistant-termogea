# Security Policy

## Supported Versions

Security updates are provided for the latest release published on `main`.

## Reporting a Vulnerability

If you find a security issue, do not open a public issue.

Report privately via:
- GitHub Security Advisories (preferred): `Security` tab in this repository

Please include:
- Affected component and version
- Steps to reproduce
- Impact assessment
- Any suggested mitigation

## Response Targets

- Initial acknowledgment: within 5 business days
- Triage and severity classification: within 10 business days
- Fix timeline: communicated after triage

## Disclosure

Please allow coordinated disclosure. A public advisory will be published after a fix is available.
